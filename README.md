## Cara Pakai

1. Baca berurutan dari foundation sampai deployment.
2. Kalau fokus ke stack tertentu, lompat ke bagian frontend/backend yang relevan.
3. Untuk produksi, jadikan checklist dan troubleshooting sebagai referensi harian.

## Catatan

Dokumentasi ini diasumsikan untuk Ubuntu Server / Debian-based Linux.
Jika kamu memakai distro lain, beberapa path atau nama service bisa berbeda.
